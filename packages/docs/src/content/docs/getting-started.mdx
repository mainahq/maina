Maina is the verification layer for AI-generated code. 18 tools. 7 languages. One command. Zero config.

## Install

<Tabs>
  <TabItem label="Global install">
    ```bash
    bun add -g @mainahq/cli
    ```
  </TabItem>
  <TabItem label="Try without installing">
    ```bash
    bunx @mainahq/cli init
    bunx @mainahq/cli verify
    ```
  </TabItem>
</Tabs>

<Aside type="tip">
**Prerequisites:** [Git](https://git-scm.com/) and [Bun](https://bun.sh/) (v1.1+). That's it.
</Aside>

## Quick Start

<Steps>

1. **Initialize your project**

   ```bash
   maina init --install
   ```

   This detects your stack, creates `.maina/` with a constitution and custom prompts, and installs missing verification tools.

   Then configure your project:

   ```bash
   maina configure
   ```

2. **Make changes and commit**

   ```bash
   git add .
   maina commit
   ```

   `maina commit` runs the full 18-tool verification pipeline — syntax guard, parallel analysis, diff-only filtering — then commits if everything passes.

3. **Run verification standalone**

   ```bash
   maina verify
   maina verify --deep     # Add AI semantic review
   maina verify --visual   # Add Playwright screenshot regression
   maina verify --cloud    # Submit to Maina Cloud
   ```

4. **Start a feature with the full workflow**

   ```bash
   maina brainstorm        # Explore the idea, create a ticket
   maina plan my-feature   # Create branch with structured spec/plan
   # ... implement ...
   maina commit            # Verify and commit
   maina pr                # Create PR with verification proof
   ```

5. **Check your setup**

   ```bash
   maina doctor
   ```

   Shows which of the 18+ tools are installed, engine health, and install commands for anything missing.

</Steps>

## Zero-Friction Layers

Maina works with zero config and scales up as you add capabilities.

| Layer | What you add | What you get |
|-------|-------------|-------------|
| **L0 — Git-native** | Nothing beyond Git + Bun | Core commands, deterministic verification, context engine |
| **L1 — Add AI** | `MAINA_API_KEY` or Ollama | AI-powered reviews, commit messages, explanations, fix generation |
| **L2 — Add tools** | Semgrep, Trivy, Secretlint, SonarQube | Deeper SAST, CVE scanning, secret detection, quality gates |
| **L3 — Add PM** | GitHub Issues | Sync to Huly, Linear, Plane, or any GitHub-syncing PM tool |

<Aside>
No accounts. No Docker. No cloud required. Everything runs locally in your terminal.
</Aside>

## What happens during `maina commit`

<Steps>

1. **Syntax guard** — Language-aware linting (Biome/ruff/go vet/clippy/dotnet format/Checkstyle/PHPStan). Under 500ms. Rejects immediately on failure.

2. **Parallel analysis** — 18+ tools run in parallel: slop, Semgrep, Trivy, Secretlint, SonarQube, Stryker, diff-cover, typecheck, consistency, ZAP, Lighthouse, AI review, and more.

3. **Diff-only filter** — Only findings on changed lines. No pre-existing noise.

4. **AI fix** (if AI configured) — Context-aware fix suggestions.

5. **Git commit** — If everything passes, your changes are committed.

6. **Workflow tracking** — Step recorded in workflow context + async RL feedback for prompt evolution.

</Steps>

## CI Integration

<Tabs>
  <TabItem label="Cloud (no local tools)">
    ```yaml
    - uses: mainahq/maina/.github/actions/verify@main
      with:
        token: ${{ secrets.MAINA_TOKEN }}
    ```
  </TabItem>
  <TabItem label="Local (on runner)">
    ```yaml
    - uses: mainahq/maina/.github/actions/verify@main
      with:
        base: main
    ```
  </TabItem>
</Tabs>

Both options run the full verification pipeline on every PR, posting findings as inline review comments. Exit codes: `0` (passed), `1` (findings), `2` (tool failure), `3` (config error). See the [CI Integration](/ci) docs for full configuration.

## Cloud Sync

Share prompts and feedback across your team:

```bash
maina login          # authenticate with Maina Cloud
maina sync pull      # get team prompts
maina sync push      # share your prompts
```

## Cloud Verification

Run the full pipeline in the cloud — no local tool installs required:

```bash
maina login              # authenticate (one-time)
maina verify --cloud     # submit diff to Maina Cloud
```

<Aside type="tip">
Results are stored as proof artifacts and returned to your terminal. The same 18+ tools run in the cloud.
See [Verify Engine — Cloud Verification](/engines/verify#cloud-verification) for details.
</Aside>
