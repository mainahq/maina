Maina is a CLI, MCP server, and skills package that proves AI-generated code is correct before it merges. This guide gets you running in under a minute.

## Install

```bash
bun add -g @mainahq/cli
```

Or try without installing:

```bash
bunx @mainahq/cli init
bunx @mainahq/cli verify
```

**Prerequisites:** [Git](https://git-scm.com/) and [Bun](https://bun.sh/) (v1.1+). That's it.

## Quick Start

### 1. Initialize

```bash
maina init --install
```

This creates the `.maina/` directory in your repo, detects your stack, and installs missing verification tools:
- `constitution.md` -- your project's non-negotiable rules
- `prompts/` -- custom AI prompt templates
- `AGENTS.md` -- cross-tool agent instructions

Then configure your project settings:

```bash
maina configure
```

### 2. Make changes and commit

```bash
# Stage your changes
git add .

# Verify and commit in one step
maina commit
```

`maina commit` runs the full 18-tool verification pipeline -- syntax guard, parallel analysis, diff-only filtering -- then commits if everything passes.

### 3. Run verification standalone

```bash
maina verify
maina verify --deep     # Add AI semantic review
maina verify --visual   # Add Playwright screenshot regression
```

### 4. Start a new feature

```bash
maina brainstorm        # Explore the idea, create a ticket
maina plan my-feature   # Create branch with structured spec/plan
# ... implement ...
maina commit            # Verify and commit
maina pr                # Create PR with verification proof
```

### 5. Check your setup

```bash
maina doctor
```

Shows which of the 18+ tools are installed, engine health, and install commands for missing tools.

## Supported Languages

Maina auto-detects your project language and adapts:

| Language | Detected from | Linter |
|----------|--------------|--------|
| TypeScript | `tsconfig.json` | Biome |
| Python | `pyproject.toml`, `requirements.txt` | ruff |
| Go | `go.mod` | go vet |
| Rust | `Cargo.toml` | clippy |
| C# (.NET) | `.sln`, `.csproj` | dotnet format |
| Java/Kotlin | `pom.xml`, `build.gradle` | Checkstyle |
| PHP | `composer.json` | PHPStan, Psalm |

## Zero-Friction Layers

Maina works with zero config and scales up as you add capabilities.

| Layer | What you add | What you get |
|-------|-------------|-------------|
| **L0 -- Git-native** | Nothing beyond Git + Bun | Core commands, deterministic verification, context engine |
| **L1 -- Add AI** | `MAINA_API_KEY` or Ollama | AI-powered reviews, commit messages, explanations, fix generation |
| **L2 -- Add tools** | Semgrep, Trivy, Secretlint, SonarQube | Deeper SAST, CVE scanning, secret detection, quality gates |
| **L3 -- Add PM** | GitHub Issues | Sync to Huly, Linear, Plane, or any GitHub-syncing PM tool |

No accounts. No Docker. No cloud. Everything runs locally in your terminal.

## What happens during `maina commit`

1. **Syntax guard** -- Language-aware linting (Biome/ruff/go vet/clippy/dotnet format/Checkstyle/PHPStan). Under 500ms. Rejects immediately on failure.
2. **Parallel analysis** -- 18+ tools run in parallel: slop, Semgrep, Trivy, Secretlint, SonarQube, Stryker, diff-cover, typecheck, consistency, ZAP, Lighthouse, AI review, and more.
3. **Diff-only filter** -- Only findings on changed lines. No pre-existing noise.
4. **AI fix** (if AI configured) -- Context-aware fix suggestions.
5. **Git commit** -- If everything passes, your changes are committed.
6. **Workflow tracking** -- Step recorded in workflow context + async RL feedback for prompt evolution.

## MCP Server

Add Maina to Claude Code, Cursor, or any MCP-compatible editor:

```json
{
  "mcpServers": {
    "maina": { "command": "maina", "args": ["--mcp"] }
  }
}
```

## CI Integration

Add Maina cloud verification to your GitHub Actions workflow:

```yaml
- uses: mainahq/maina/.github/actions/verify@main
  with:
    token: ${{ secrets.MAINA_TOKEN }}
```

Or run locally on the runner (no cloud account needed):

```yaml
- uses: mainahq/maina/.github/actions/verify@main
  with:
    base: main
```

Both options run the full verification pipeline on every PR, posting findings as inline review comments. Exit codes: `0` (passed), `1` (findings), `2` (tool failure), `3` (config error). See the [CI Integration](/maina/ci) docs for full configuration.

## Cloud Sync

Share prompts and feedback across your team with Maina Cloud:

```bash
maina login          # authenticate with Maina Cloud
maina sync pull      # get team prompts
maina sync push      # share your prompts
```

Team prompts, constitution updates, and A/B test results sync automatically. No self-hosting required.

## Cloud Verification

Run the full verification pipeline in the cloud instead of locally. Useful for CI, teams with heterogeneous environments, or when you want results without installing 18+ tools.

```bash
maina login              # authenticate (one-time)
maina verify --cloud     # submit diff to Maina Cloud for verification
```

Your diff is submitted to the hosted pipeline. Results are stored as proof artifacts in R2 and returned to your terminal. The same 18+ tools run in the cloud -- no local installs required.

See the [Verify Engine -- Cloud Verification](/maina/engines/verify#cloud-verification) docs for architecture details.
