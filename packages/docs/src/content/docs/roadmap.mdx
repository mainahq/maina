Maina is open source (Apache 2.0) and actively developed. Here is what is shipped and planned.

## Shipped (v0.3.0)

- **25 CLI commands** including `maina brainstorm` as the workflow starting point
- **16-tool verification pipeline** -- Biome, ruff, go vet, clippy, dotnet format, Checkstyle, Semgrep, Trivy, Secretlint, SonarQube, Stryker, diff-cover, AI review, slop detection, visual regression, SpotBugs/PMD
- **6 languages** -- TypeScript, Python, Go, Rust, C#/.NET, Java/Kotlin
- **Structured AI delegation** -- `---MAINA_AI_REQUEST---` stdout protocol for any MCP-compatible host
- **Visual verification** -- Playwright screenshot regression testing
- **Workflow context** -- rolling summary forwarded between lifecycle steps
- **Background RL** -- async feedback recording at each workflow step with post-workflow analysis
- **Verification proof** -- `maina pr` includes verification artifacts in the PR body
- **`maina brainstorm`** -- interactive idea exploration that generates structured GitHub issues
- **`maina configure`** -- interactive constitution, conventions, tools, visual URLs
- **`maina init --install`** -- auto-install missing verification tools
- **8 MCP tools** for IDE integration
- **5 cross-platform skills** for Claude Code, Cursor, Codex, Gemini CLI
- **1017+ tests**, 100% benchmark (matches SpecKit)

## Shipped (v0.3.x) -- Hardening

- **Built-in type checking** -- `tsc --noEmit` runs as a verify step with zero extra installs
- **Cross-function consistency check** -- AST-based deterministic check that related functions use consistent patterns
- **Honest "0 tools" warning** -- Reports a warning and suggests `maina init --install` instead of false "0 findings, passed"
- **`maina init` auto-configures Biome** -- Every initialized project gets at least one real linter out of the box
- **`--auto` flags for `maina spec` and `maina design`** -- Skip interactive prompts for CI and scripting workflows
- **Post-workflow RL trace analysis** -- Analyzes complete traces to propose prompt improvements fed into `maina learn`
- **1036+ tests** across 95 files

## v2.0.0 -- Enterprise

Built on the same Workkit foundation, hardened for enterprise.

### Security + compliance

- **Encryption at rest** -- @workkit/crypto (AES-256-GCM) for all stored data: feedback, prompts, verification results, cache
- **Per-org encryption keys** -- Key rotation support, managed via enterprise admin API
- **SSO / SAML** -- @workkit/auth extended with SAML 2.0 and OIDC for enterprise IdPs
- **Audit logging** -- Every verification, prompt change, feedback event logged immutably, SIEM export (Splunk, Datadog)
- **SOC 2 Type II** -- Controls documented, penetration tested, data residency options (EU, US, APAC)

### Air-gapped deployment

- **Self-hosted option** -- Docker compose stack without Cloudflare dependencies: D1→SQLite/Postgres, KV→Redis, R2→S3/MinIO, Queue→BullMQ
- **Ollama as default** -- No external AI API calls required
- **@workkit/testing mocks** -- Used in CI for enterprise deployment tests

### Advanced features

- **Custom model fine-tuning** -- Export team feedback as training data, fine-tune via @workkit/ai-gateway
- **Chrome extension** -- Review AI-generated code in GitHub PRs
- **`transformContext` hooks** -- Programmatic context manipulation
- **Self-extending agent** -- Maina proposes its own extensions based on usage patterns

## Contributing

Maina is open source and welcomes contributions.

### Getting started

```bash
git clone https://github.com/mainahq/maina
cd maina
bun install
bun run build
bun run test
```

### Development commands

```bash
bun run dev          # Dev mode with watch
bun run check        # Biome lint + format check
bun run typecheck    # tsc --noEmit
bun run verify       # Full verification: check + typecheck + test
```

### Conventions

- **Conventional commits** with scopes: `cli`, `core`, `mcp`, `skills`, `docs`, `ci`
- **TDD always** -- write tests first, watch them fail, implement, watch them pass
- **Result pattern** -- `Result<T, E>` for error handling, never throw
- **Bun only** -- Bun runtime, bun:test, bun:sqlite. Not Node.js, not Jest, not Vitest.

### Where to contribute

- **New language profiles** -- Add a `LanguageProfile` in `packages/core/src/language/profile.ts`
- **New analysis tools** -- Add a tool adapter in `packages/core/src/verify/`
- **Linter parsers** -- Add output parsers in `packages/core/src/verify/linters/`
- **Skills** -- Write new SKILL.md files in `packages/skills/`
- **Documentation** -- Improve these docs in `packages/docs/`
- **Bug reports and feature requests** -- Open a GitHub Issue

### Dogfooding

Every commit to Maina goes through `maina commit`. If you contribute, use `maina commit` for your commits too. If a gate is noisy, fix the gate -- do not skip it.
