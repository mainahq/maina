## Bugs found + fixed in QA branch (PR #221)

10 review-comment follow-ups + 1 new QA finding:

### Security

- `adopt.ts` + `scan/tree-sitter.ts` walkers followed symlinks out of `cwd`. Switched `statSync → lstatSync` and added symlink skip — a `.cursor/rules → /etc` symlink can no longer be read.

### Correctness bugs

- `context.ts --output`: `mkdirSync(join(resolved.path, ".."))` creates a directory with the output file's name. Switched to `dirname(resolved.path)`.
- `wiki/init.ts --background`: `bun run maina` fails in user repos (no `maina` script). Switched to `Bun.which("maina")` with `process.execPath` + `argv[1]` fallback.
- `wiki/status.ts` staleness: compared `startedAt` (frozen) instead of file mtime (updated on progress). Fixed.
- `skills-deploy.ts`: `new URL(".", import.meta.url).pathname` is percent-encoded on paths with spaces. Switched to `fileURLToPath`.
- `doctor.ts` `claudeSettings`: reported `false` for users with global-only Claude registration. Now `true` for any scope.
- `doctor.ts --fix`: didn't recompute mcpHealth after fix ran. Now re-queries.
- `tailor.ts validateConstitution`: only checked heading presence; accepted paraphrased workflow sections. Now compares body whitespace-normalised against the template.
- `mcp/server.ts allTools: true`: stripped DeepWiki compat tools. Now gated on `MAINA_MCP_STRICT_TEN=1`.
- `confirm.ts openEditor`: no-op stub. Now spawns `$EDITOR` with temp-file round-trip.
- `scan/lint-config.ts`: advertised `ruff.toml` support but never parsed it. Removed from candidate list.
- `buildHostFallbackConstitution` (new QA finding): stub lacked `## Maina Workflow` + `## File Layout`, violating G3/G7 inside the host-delegation path. Now appends both sections via the same renderers.

### Hygiene

- `.gitignore` now excludes `packages/**/__tests__/tmp-*/` and language-detect `__fixtures__/`. Two stray `tmp-wiki-*` dirs were accidentally committed by Wave 4 tests and subsequently cleaned up.
