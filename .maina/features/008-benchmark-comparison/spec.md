# Feature: Sprint 1-9 Benchmark — Claude+Superpowers vs Claude+Maina

## Problem Statement

Maina was built over 10 sprints (0-9) to replace ad-hoc AI-assisted development (Claude + Superpowers skills) with a verification-first pipeline. We need hard numbers to prove the value: what did maina actually improve?

## Target User

- Maina team evaluating ROI of the tooling investment
- Potential adopters who want evidence before committing

## Success Criteria

- [ ] Side-by-side comparison table across all 10 sprints
- [ ] Quantified improvement in at least 3 categories
- [ ] Data gaps documented honestly (what we can't measure)
- [ ] Comparison exportable as markdown for README/docs

## Scope

### In Scope

- Metrics we can derive from git history, stats.db, feedback.db, benchmarks
- Qualitative observations from the development process
- Sprint-by-sprint progression showing maina bootstrapping itself

### Out of Scope

- A/B testing with separate teams (single developer project)
- Cost analysis (API spend tracking not implemented)
